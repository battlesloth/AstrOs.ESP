# Firmware-version compatibility gating — Phase 1 (firmware)

## Context

AstrOs.Server has no way today to know what firmware version is running on each ESP32 controller. This is the firmware-side phase of a cross-repo feature: each ESP will start reporting its firmware version on every `POLL_ACK` so the server can hard-code a `MINIMUM_FIRMWARE_VERSION` and gate UI based on it.

After this phase merges, controllers will report their firmware version. The server (still running pre-Phase 2 code) ignores the extra field harmlessly; nothing visibly changes for the end user yet. Phase 2 (server + Vue) lands separately in the AstrOs.Server repo on branch `feature/firmware-version-gating` and is what activates the new visual states.

## Decisions (confirmed with user)

- **Wire format**: extend `POLL_ACK` payload only; leave `REGISTRATION_SYNC_ACK` alone. Each ESP responds to its own poll, so each controller's own POLL_ACK is the natural carrier — no ESPNOW mesh changes. Brief unknown-version window after registration is acceptable; the first poll cycle resolves it.
- **Initial version**: bump `VERSION` to `1.2.0`. Phase 2 will hard-code `MINIMUM_FIRMWARE_VERSION = '1.2.0'`, so this firmware will be the first compatible build.
- **Source of truth**: use `AstrOsConstants::Version` (auto-generated by `scripts/version_gen.py` from the `VERSION` file). No new constants needed.

## Wire-format change

`POLL_ACK` payload today: `mac<US>name<US>fingerprint`
`POLL_ACK` payload after: `mac<US>name<US>fingerprint<US>version`

`<US>` = `0x1F`. Append-only.

## Files to modify

- `VERSION` — bump from `1.1.0` → `1.2.0`. The PlatformIO pre-build hook (`scripts/version_gen.py`) regenerates `lib_native/AstrOsUtility/src/version_generated.hpp` automatically.
- `lib_native/AstrOsMessaging/src/AstrOsSerialMessageService.cpp` — extend `getPollAck()` (lines ~124–131):
  ```cpp
  ss << macAddress << UNIT_SEPARATOR << controller << UNIT_SEPARATOR
     << fingerprint << UNIT_SEPARATOR << AstrOsConstants::Version;
  ```
  Add `#include "AstrOsConstants.hpp"` at the top of the file (verify it isn't already pulled in transitively).
- `test/test_native/astros_serial_messages_tests.cpp` — update the POLL_ACK test (search for `getPollAck`) so the expected string includes the new `<US>version` suffix.

## Out of scope

- Adding firmware version to `REGISTRATION_SYNC_ACK` (would require ESPNOW-level peer→master version exchange; deferred — POLL_ACK fills the gap within one poll cycle).
- Server-side changes — those land in Phase 2 in the AstrOs.Server repo (`feature/firmware-version-gating`).
